# 클래스 생성 및 통합 워크플로우 (Create Class Do)

이 워크플로우는 `KeyEventMonitor` 리팩토링 사례를 기반으로, 새로운 클래스를 안티그래비티(Antigravity) 프로젝트 표준에 맞춰 생성하고 통합하는 절차를 안내합니다.

## 1. 분석 및 필요성 확립 (Analysis & Justification)
- [ ] **소스 분석**: 분리하거나 새로 생성할 기능의 범위를 기존 코드에서 분석합니다.
- [ ] **필요성 확립**: 다음 기준에 따라 클래스 생성의 명분을 확립합니다.
    - **통일성 (Consistency)**: 프로젝트의 아키텍처 패턴과 일치하는가?
    - **재사용성 (Reusability)**: 다른 부분에서도 사용할 수 있는가?
    - **관리 용이성 (Maintainability)**: 코드를 더 작고 이해하기 쉬운 단위로 나누는가?
- [ ] **계획 수립**: `task.md` 또는 `implementation_plan.md`에 계획을 기록합니다.

## 2. 클래스 생성 (Class Creation)
- [ ] **파일 생성**: 확립된 명분에 따라 적절한 위치(예: `Core/Managers`, `Data`, `Utils` 등)에 새 클래스 파일을 생성합니다.
- [ ] **구현**: 기능을 구현하고 필요한 프로토콜(예: `Delegate`)을 정의합니다.

## 3. 이모지 등록 (Emoji Registration)
- [ ] **워크플로우 실행**: 주석으로 `// turbo`를 달고 다음 명령어를 실행하여 새 클래스에 대한 이모지를 생성하고 등록합니다.
    ```bash
    /register-class-emoji
    ```
- [ ] **확인**: `_doc_work/reference_EMOJI.md` 또는 관련 설정 파일에 이모지가 올바르게 등록되었는지 확인합니다.

## 4. 로그 적용 (Apply Emoji to Logs)
- [ ] **로깅**: 새로 생성된 클래스의 로그 메시지에 등록된 이모지를 적용합니다.
    - 예: `logD("✨ [NewClass] ...")`
- [ ] **확인**: 앱을 실행하고 로그를 확인하여 이모지가 정상적으로 출력되는지 검증합니다.

## 5. 코드 정리 (Cleanup Obsolete Code)
- [ ] **리팩토링**: 생성된 클래스를 사용하는 코드로 기존 코드를 교체합니다.
- [ ] **제거**: 더 이상 필요 없어진 기존 코드(메서드, 변수 등)를 주석 처리하거나 삭제합니다.

## 6. 설계 문서 검토 및 재사용 (Review & Reuse)
- [ ] **문서 검토**: `_doc_design/` 폴더의 문서를 검토하여, 새로 만든 클래스와 유사하거나 중복되는 기존 객체가 있는지 확인합니다.
- [ ] **재사용 적용**: 재사용 가능한 부분이 있다면 적용하고, 통합합니다.

## 7. 불필요한 객체 제거 (Remove Obsolete Objects)
- [ ] **객체 제거**: 재사용으로 인해 필요 없어진 기존 객체나 파일을 제거합니다.
- [ ] **이모지 업데이트**: 제거된 객체가 있다면 이모지 등록을 업데이트합니다.
    ```bash
    /register-class-emoji
    ```

## 8. 문서 업데이트 (Update Documentation)
- [ ] **문서 반영**: `_doc_design/` 폴더의 관련 문서(예: `design_ARCHITECTURE.md`, 클래스 다이어그램 등)에 새로 추가된 객체를 반영합니다.
- [ ] **완료**: 변경 사항을 커밋하고 완료합니다.
