description: "이슈 관리 및 자동화 워크플로우 (분석 -> 등록/이동 -> 해결)"
---

**역할**: `Issue.md`의 상태를 분석하여 다음에 무엇을 해야 할지 **자동으로 결정하고 실행**합니다.

## 1. 이슈 상태 분석 (Analyze Issue Status)
`Issue.md` 파일을 읽고 아래 우선순위에 따라 작업을 수행합니다.

### **Priority 1: 이슈 후보 처리 (Process Candidates)**
- **대상**: `# 🌱 이슈후보` 섹션
- **조건**: 해당 섹션에 내용이 있는 경우.
- **행동**: **`/issue-reg` 워크플로우 실행**.
    - 후보 내용을 분석하여 중요도(`📕 중요`, `📙 일반`, `📗 선택`)에 따라 적절한 섹션으로 등록/이동합니다.
    - 구현이 필요 없는 항목은 정리하거나 일반 이슈로 분류합니다.
    - 너무 큰 이슈는 서브 이슈(예: `Issue123_1`)로 분리합니다.
    - **중요**: 부모 이슈가 이미 **완료(Closed)**된 경우, 서브 이슈는 독립적인 이슈로 취급하여 활성 섹션(`📙 일반`, `📕 중요`)에 등록합니다.

### **Priority 2: 진행 중인 이슈 해결 (Fix In-Progress)**
- **대상**: `# 🚧 진행중` 섹션
- **조건**: 해당 섹션에 이슈가 할당되어 있는 경우.
- **행동**: **`/issue-fix` 워크플로우 실행**.
    - 이미 진행 중인 작업이 있으므로, 해당 이슈의 해결 프로세스(분석->구현->검증)를 이어서 진행합니다.

### **Priority 3: 중요 이슈 착수 (Start Important)**
- **대상**: `# 📕 중요` 섹션
- **조건**: 진행 중인 이슈가 없고, 이슈 후보도 처리된 상태일 때.
- **행동**:
    1. `# 📕 중요` 섹션의 **가장 상위(첫 번째) 이슈**를 선택합니다.
    2. 해당 이슈를 `# 🚧 진행중` 섹션으로 **이동**시킵니다.
    3. **`/issue-fix` 워크플로우 실행**하여 작업을 시작합니다.

### **Priority 4: 일반 이슈 착수 (Start General)**
- **대상**: `# 📙 일반` 섹션
- **조건**: 중요 이슈도 없는 경우.
- **행동**:
    1. `# 📙 일반` 섹션의 첫 번째 이슈를 선택합니다.
    2. 해당 이슈를 `# 🚧 진행중` 섹션으로 이동시킵니다.
    3. **`/issue-fix` 워크플로우 실행**합니다.
