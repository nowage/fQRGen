# `lib` 개선 (Library Enhance) 워크플로우

`lib-apply`를 통해 대상 모듈에 적용된 코드는, 이후 앱 개발 과정에서 Xcode 프로젝트 내에서 꾸준히 개선될 수 있습니다. 
이 경우 `lib/` 폴더 내의 원본 코드가 '구 버전' 방치 상태가 되는 것을 방지하기 위해, **앱(본체)의 최신 코드를 다시 `lib/` 폴더로 가져와 최신화**하는 워크플로우입니다.

향후 기능 추가나 재검증이 필요할 때, 이 최신화된 `lib/` 스크립트를 기반으로 `lib-apply` 워크플로우를 문제없이 재수행할 수 있도록 합니다.

## 1. 매핑 확인 (Verify Mapping)
어느 `lib` 스크립트가 어느 앱 모듈 스크립트와 연결되어 있는지, 해당 라이브러리 폴더의 `README.md`에서 확인합니다.

> ⚠️ **중요:** 각 `lib/[모듈명]/README.md` 에는 해당 라이브러리와 본체 대상 코드의 매핑 관계가 명확히 명시되어 있어야 합니다. (예: `lib/import/README.md`)

## 2. 최신 코드 가져오기 (Bring Latest Code to Lib)
앱(fQRGen) 내부에 반영되어 수정·개선된 최신 로직을 `lib/`의 스크립트로 덮어쓰거나 이식하여 현행화합니다.
의존성 문제 없이 `lib` 내 스크립트 단독으로도 테스트 가능한 구조를 유지하도록 가급적 주의합니다.

## 3. 독립 실행 테스트 (Standalone Trial)
이식된 최신 코드가 독립적인 샌드박스(lib 내부) 환경에서도 정상 작동하는지 테스트합니다.
- `swift lib/...` 등의 방식으로 실행
- 필요 시 테스트를 위한 Mock Data나 의존 파일들 재정비

## 4. README.md 최신화 (Update Library Docs)
코드 개선에 따라 문서나 사용 방법이 변경되었다면, 해당 `lib/[모듈명]/README.md` 파일을 업데이트하여 연결점과 변경 이력을 명시합니다.

## 5. 저장 (Commit)
`lib/` 폴더 내 파일 최신화 작업 내역을 Git에 commit 하여 상태를 고정시킵니다.
