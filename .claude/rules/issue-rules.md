* 규칙0 (프로세스): 사용자가 승인하기 전까지는 완료 섹션으로 이동 불가하며, 작업 시작 시 `# 🚧 진행중`, 완료 시 `# 🏁 완료-해결순`으로 이동합니다.
* 규칙1 (작성 언어): 이슈 제목, 목적, 상세 내용 및 결과 설명(Walkthrough)은 모두 **한국어**로 작성합니다. (기술 용어 제외)
* 규칙2 (이슈 포맷): 이슈 번호는 붙여쓰기(Issue999)하며, `* 목적`, `* 상세`, `* 구현 명세`(로직/검증 포함) 구조를 준수합니다. 서브 이슈는 `_`로 구분(Issue2_3)하고 부모 이슈 바로 아래 배치합니다.
* 규칙3 (커밋 정책): 완료된 이슈는 `issue-manager`가 자동으로 제목에 해시를 기록합니다. 본문(`* 상세` 등)에 중복으로 기록하지 않습니다. 서브 이슈는 부모 이슈 종결 시 함께 처리됩니다.
* 규칙4 (서브 이슈): 복잡한 기능은 서브 이슈로 분리하며, 서브 이슈 완료 시 '완료' 표시만 하고 부모 이슈가 종결될 때 함께 이동합니다.
* 규칙5 (정리): 이슈 등록 시 `🌱 이슈후보`의 중복 항목은 삭제.
* 규칙6 (동기화): 서브 이슈는  항상 메인 이슈 하위 개요에 위치해야 합니다. (활성/완료 상태 무관)

